# AI-RULES.md — My SaaS App

> Tool-agnostic AI coding rules. Works with any AI coding assistant.
> Generated by ai-rules sync engine.

## Git

- Branch: `feature/{YYMMDD}-{description}`
- Commit: `{type}({scope}): {description}` (conventional commits)
- Never commit to main/master/develop directly
- Never: rebase, force push, auto-resolve conflicts

## Code

- Frontend: hooks + React Query (no direct fetch in components)
- Backend: Controller → Service → Repository (3-layer)
- No hardcoded secrets, no PII in logs
- No `@ts-ignore`, `eslint-disable`

## Security

- R0 (safe): local changes, narrow scope → auto
- R1 (caution): reversible but costly → ask user
- R2 (danger): data loss, external changes → user executes directly
- STRIDE check on new features

## Workflow

- Plan before implementing (bug fixes, features, API changes)
- Lint + type check before every commit
- Agent order: planner → builder → reviewer
