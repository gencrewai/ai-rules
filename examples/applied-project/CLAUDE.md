# 01-git — Git Rules

## Banned Commands (CRITICAL)

- **Never**: `rebase`, `merge` (except ff-only), `checkout --ours/--theirs`, `push -f/--force`
- **Allowed**: `status`/`log`/`diff`/`fetch`, `checkout -b`, `add`/`commit`/`push origin {branch}`
- On conflict: abort immediately, report to user

## Branch Rules

- Format: `feature/{YYMMDD}-{description}`
- Always work on feature branches — never commit to main/master/develop
- Stale branch: 3+ days or 50+ commits behind → warn user

## Commit Rules

- Format: `{type}({scope}): {description}` (conventional commits)
- Co-Authored-By header required

## Commit & Push Process

1. Branch protection check (main/master/develop → block)
2. Source validation (lint-staged via pre-commit hook)
3. Diff report: changed files + line count
4. Mode-based action (manual/auto/auto-push)

# 02-code — Code Architecture

## Hard Bans

### Frontend (React)
- No `fetch`/`axios` in components → hooks + React Query
- `dangerouslySetInnerHTML` requires `DOMPurify.sanitize()`
- No hardcoded colors → CSS variables or Tailwind semantic classes
- Page components must use `lazy()` import

### Backend (FastAPI)
- No DB in Controller → Controller → Service → Repository
- All endpoints require auth middleware
- All endpoints require `response_model`
- New FK columns require index

### Common
- No hardcoded secrets
- No PII in production logs
- No repo-wide scanning
