# engine/ — Multi-Project Sync Tooling (Tier 2)

**If you want to deploy the same rules across multiple projects**, use this tool.
It assembles `core/` rules via profile YAMLs and generates the appropriate output for each project.

## Quick Start

```bash
cd engine
npm install

# Create a new project profile
npm run new -- my-project

# Generate rules (saved to output/)
npm run sync

# Apply to actual projects
npm run sync:apply

# Target a specific project only
node scripts/sync.mjs --apply --project my-project

# Validate
npm run validate
```

## Adapters (Supported Tools)

| Tool | Output File | Status |
|------|------------|--------|
| Claude Code | `CLAUDE.md` | ✅ |
| Cursor | `.cursor/rules/*.mdc` | ✅ |
| Windsurf | `.windsurfrules` | ✅ |
| Plain (fallback) | `AI-RULES.md` | ✅ |

Adding a new tool: Create `adapters/{tool-name}.mjs` and enable it in the profile.

## Profile System

Each project defines its rule composition via a YAML profile.

```yaml
# profiles/my-project.yaml
project: my-project
target_path: /path/to/my-project

tools:
  claude-code: { enabled: true, output: CLAUDE.md }
  cursor: { enabled: true, output: .cursor/rules/ }

core: [01-git, 02-code, 03-security, 04-workflow]
extensions: [my-backend-rules]
agents: { include: [planner, builder, reviewer] }
```

→ Examples: [examples/profiles/](../examples/profiles/)

## Governance Engine

**Cross-validates** code generated by AI agents.

| Preset | Target | Validation Level |
|--------|--------|-----------------|
| `solo` | Solo developer | Basic validation |
| `small-team` | 2-5 person team | Structure + conventions |
| `saas` | Production service | Structure + conventions + domain + security |

## Editing Rules

1. Edit files in `../core/rules/` or extension files
2. `npm run sync` → Generate output
3. `npm run sync:apply` → Apply to projects

**Never edit `output/` files or a project's CLAUDE.md directly.**
